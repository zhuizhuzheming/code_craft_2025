大体思路：
一开始想用动态规划的，后来发现超时，虽然理论上来说可以拿到1kw。<br>
于是后来改用了贪心法，对于无法读取的任务则直接舍弃，结果光用这个也还是超时<br>
对于多次读取的任务，采用缓存的形式存取，在某一定的时间内选择缓存直接读取，过了一定的时间之后再进行上报。<br>
由于题目中说了每一个tag有各自的生命周期，因此我在前面的12600个时间片内进行学习它们的生命周期，为后面的缓存策略提供借鉴。
